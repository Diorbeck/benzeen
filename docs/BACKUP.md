# Резервное копирование и восстановление БД

Ежедневный бэкап продакшн-базы PostgreSQL выполняется через GitHub Actions:
`.github/workflows/db-backup.yml`.

## Что делает workflow

- Запускается каждый день в **02:30 UTC** (≈07:30 по Ташкенту) и вручную
  (вкладка **Actions → Daily DB Backup → Run workflow**).
- Делает `pg_dump -Fc` (custom format, сжатый, поддерживает выборочное
  восстановление), затем `gzip`.
- Сохраняет дамп как **приватный артефакт** GitHub Actions с хранением
  **30 дней**.
- Если заданы AWS-секреты — дополнительно копирует дамп в S3.

Строка подключения передаётся через секрет и **никогда не печатается в лог**.

## Необходимые секреты репозитория

Settings → Secrets and variables → Actions → **New repository secret**:

| Секрет | Обязательность | Назначение |
| --- | --- | --- |
| `DATABASE_URL` | **обязательно** | Полная строка подключения к прод-базе |
| `AWS_ACCESS_KEY_ID` | опционально | Включает выгрузку в S3 |
| `AWS_SECRET_ACCESS_KEY` | опционально | — |
| `AWS_REGION` | опционально | напр. `eu-central-1` |
| `BACKUP_S3_BUCKET` | опционально | напр. `s3://my-bucket/benzeen-backups` |

S3-выгрузка включается только если заданы `AWS_ACCESS_KEY_ID` и
`BACKUP_S3_BUCKET`.

## Как скачать бэкап

1. **Actions → Daily DB Backup** → выберите нужный запуск.
2. В разделе **Artifacts** скачайте файл `benzeen-YYYYMMDD-HHMMSS.dump.gz`.

## Как восстановить — ТОЛЬКО НА КОПИИ

> ⚠️ **Никогда не восстанавливайте поверх боевой базы.** Сначала создайте
> отдельную (пустую) базу-копию и проверьте дамп на ней.

```bash
# 1. Распаковать
gunzip benzeen-YYYYMMDD-HHMMSS.dump.gz   # → benzeen-YYYYMMDD-HHMMSS.dump

# 2. Создать ПУСТУЮ копию (другое имя/инстанс, НЕ прод!)
createdb -h <host> -U <user> benzeen_restore_test

# 3. Восстановить в копию
pg_restore --no-owner --no-privileges \
  -h <host> -U <user> -d benzeen_restore_test \
  benzeen-YYYYMMDD-HHMMSS.dump

# 4. Проверить данные в benzeen_restore_test, и только потом —
#    при реальной аварии — повторить восстановление на проде
#    осознанно, вручную, с предварительным бэкапом текущего состояния.
```

## Рекомендации

- Раз в месяц делайте «учебное восстановление» на копии — бэкап без
  проверенного восстановления не считается рабочим.
- Если используете Railway/managed-Postgres, держите включёнными и их
  встроенные снапшоты — это дополнительный, независимый уровень защиты.
